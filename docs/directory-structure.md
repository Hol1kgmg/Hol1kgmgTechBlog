# ディレクトリ構造

プロジェクトはFeature-Sliced Design (FSD)の原則に従って構成されています。

```
src/
├── app/                    # アプリケーション全体の設定、スタイル、Provider
│   ├── layout.tsx         # ルートレイアウト
│   ├── page.tsx          # ルートページ
│   └── globals.css       # グローバルスタイル
│
├── processes/             # 複雑なページ間プロセス
│   └── auth/             # 認証関連のプロセス
│
├── pages/                # ページコンポーネント
│   ├── dashboard/        # ダッシュボードページ
│   ├── articles/         # 記事一覧・詳細ページ
│   ├── profile/         # プロフィールページ
│   └── mini-profile/    # ミニプロフィールページ
│
├── widgets/              # 複合的なUIブロック
│   ├── Header/          # ヘッダーウィジェット
│   └── Footer/          # フッターウィジェット
│
├── features/            # ユーザーインタラクション機能
│   ├── articleSearch/   # 記事検索機能
│   └── articleCreate/   # 記事作成機能
│
├── entities/            # ビジネスエンティティ
│   ├── article/        # 記事エンティティ
│   └── user/           # ユーザーエンティティ
│
└── shared/             # 再利用可能な機能
    ├── api/           # APIクライアント
    ├── config/        # 設定
    ├── lib/           # ユーティリティ関数
    └── ui/            # UIコンポーネント
```

## レイヤーの説明

### app
アプリケーション全体の設定、スタイル、Providerなどを含むレイヤーです。
- ルートレイアウト
- グローバルスタイル
- アプリケーションの設定
- プロバイダー

### processes
複数のページにまたがる複雑なプロセスを管理するレイヤーです。
- 認証フロー
- マルチステップフォーム
- 複雑なワークフロー

### pages
エンティティ、フィーチャー、ウィジェットを組み合わせてページを構成するレイヤーです。
- ダッシュボードページ
- 記事一覧ページ
- 記事詳細ページ
- プロフィールページ
- ミニプロフィールページ

### widgets
エンティティとフィーチャーを意味のあるブロックに結合するレイヤーです。
- ヘッダー
- フッター
- サイドバー
- 記事リスト

### features
ユーザーとのインタラクションや、ビジネス価値をもたらす機能を提供するレイヤーです。
- 記事検索
- 記事作成
- コメント投稿
- いいね機能

### entities
ビジネスドメインのエンティティを定義するレイヤーです。
- 記事
- ユーザー
- コメント
- カテゴリー

### shared
プロジェクト全体で再利用可能な機能を提供するレイヤーです。
- UIコンポーネント
- APIクライアント
- ユーティリティ関数
- 型定義
- 定数
