# ヘッダー・フッターコンポーネント設計

## 概要

このドキュメントでは、アプリケーション全体で使用される共通のヘッダーとフッターコンポーネントの設計方針と実装ガイドラインを定義します。

## ディレクトリ構造

```
frontend/src/shared/ui/
├── Header/
│   ├── Header.tsx
│   └── index.ts
└── Footer/
    ├── Footer.tsx
    └── index.ts
```

## コンポーネント一覧

### 1. Header

#### 概要
- アプリケーションのヘッダーコンポーネント
- タイトル表示と検索機能を提供
- すべてのページで一貫したナビゲーション体験を提供

#### Props
```typescript
interface HeaderProps {
  title: string;
  showSearch?: boolean;
}
```

#### 使用例
```tsx
<Header title="ダッシュボード" showSearch={true} />
```

### 2. Footer

#### 概要
- アプリケーションのフッターコンポーネント
- コピーライト表示とリンク集を提供
- すべてのページで一貫したフッター情報を表示

#### Props
```typescript
interface FooterProps {
  copyright?: string;
  links?: Array<{
    label: string;
    href: string;
  }>;
}
```

#### 使用例
```tsx
<Footer 
  copyright="© 2024 Hol1kgmgTechBlog"
  links={[
    { label: "利用規約", href: "/terms" },
    { label: "プライバシーポリシー", href: "/privacy" }
  ]}
/>
```

## 設計原則

### 1. 一貫性
- すべてのページで同じデザインと機能を提供
- ユーザーエクスペリエンスの統一性を維持

### 2. レスポンシブ対応
- モバイル、タブレット、デスクトップに対応
- 画面サイズに応じた適切なレイアウト調整

### 3. アクセシビリティ
- 適切なARIA属性の使用
- キーボードナビゲーションのサポート
- スクリーンリーダー対応

## 実装ガイドライン

### 1. コンポーネント作成手順
1. コンポーネントの役割と責務を明確に定義
2. 必要なPropsを設計
3. 基本的なスタイリングを実装
4. アクセシビリティ対応
5. レスポンシブ対応の実装

### 2. 命名規則
- コンポーネント名はPascalCase
- ファイル名はコンポーネント名と一致
- Propsの型定義は`コンポーネント名Props`の形式

### 3. スタイリング
- Tailwind CSSを使用
- ダークテーマに最適化
- カスタマイズ可能なclassNameプロパティ

## メンテナンス

### 1. バージョン管理
