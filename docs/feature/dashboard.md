# ダッシュボード画面

## 概要
ダッシュボード画面は、ユーザーがアプリケーションの主要な情報や機能にアクセスするための中心的なインターフェースです。

## 機能一覧

### 1. 記事管理
- 最近作成した記事の一覧表示
- 下書き中の記事の表示
- 記事の編集・削除機能
- 新規記事作成へのクイックアクセス

### 2. アクティビティフィード
- 最近のアクティビティ表示
- コメントやいいねの通知
- フォロワーからのアクション表示

### 3. 統計情報
- 記事の閲覧数
- フォロワー数
- いいね数
- コメント数

### 4. クイックアクション
- 新規記事作成
- プロフィール編集
- 設定へのアクセス

### 5. Dockナビゲーション
- 画面下部に固定されたドック型ナビゲーション
- 主要機能へのクイックアクセス
- ホバー時の拡大アニメーション
- カスタマイズ可能なアイコンとラベル

## 画面構成

### レイアウト
```
+------------------------+
|        ヘッダー        |
+------------------------+
| サイドバー | メイン    |
|           | コンテンツ |
|           |           |
|           |           |
+-----------+-----------+
|      Dock ナビゲーション    |
+------------------------+
```

### コンポーネント構成
1. ヘッダー
   - ユーザー情報
   - ナビゲーションメニュー
   - 検索バー

2. サイドバー
   - メニューリスト
   - クイックアクション
   - 統計情報

3. メインコンテンツ
   - 記事一覧
   - アクティビティフィード
   - 通知

4. Dockナビゲーション
   - ホーム
   - 記事作成
   - プロフィール
   - 設定
   - 通知

## 技術的な実装

### 使用コンポーネント
- `pages/dashboard/page.tsx`: メインページコンポーネント
- `widgets/DashboardHeader`: ヘッダーウィジェット
- `widgets/DashboardSidebar`: サイドバーウィジェット
- `widgets/DockNavigation`: ドック型ナビゲーションウィジェット
- `features/articleList`: 記事一覧機能
- `features/activityFeed`: アクティビティフィード機能

### データフロー
1. ページロード時に必要なデータを取得
   - ユーザー情報
   - 記事一覧
   - アクティビティデータ
   - 統計情報

2. リアルタイム更新
   - WebSocketを使用した通知のリアルタイム更新
   - 定期的な統計情報の更新

## アクセス制御
- 認証済みユーザーのみアクセス可能
- ユーザーロールに基づく機能の制限
