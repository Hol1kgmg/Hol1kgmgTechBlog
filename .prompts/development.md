## 開発環境のセットアップ

### 必要条件
- Node.js (v18以上)
- npm または yarn
- Git

### インストール手順

1. リポジトリのクローン
```bash
git clone [リポジトリURL]
cd [プロジェクト名]
```

2. 依存関係のインストール
```bash
npm install
# または
yarn install
```

3. 開発サーバーの起動
```bash
npm run dev
# または
yarn dev
```

## 開発フロー

1. 新しい機能開発は新しいブランチで行います
2. コミットメッセージは明確で具体的に記述してください
3. プルリクエストを作成する前に、コードの品質チェックを実行してください

## TypeScript開発フロー

1. テスト駆動開発（TDD）
   - テストを先に書く
   - 実装を書く
   - リファクタリング

2. コード品質
   - ESLintとPrettierでコードスタイルを統一
   - TypeScriptの型チェックを厳格に
   - テストカバレッジを維持

3. ドキュメント
   - JSDocコメントでAPIを文書化
   - 複雑なロジックには説明コメントを追加
   - 変更履歴を記録

4. エラーハンドリング
   - 型安全なエラー処理
   - 適切なエラーメッセージ
   - エラーの伝播を制御

## トラブルシューティング

問題が発生した場合は、以下の手順を試してください：

1. `node_modules`を削除して依存関係を再インストール
2. キャッシュをクリア
3. 開発サーバーを再起動
